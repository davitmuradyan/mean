Guides to follow before submitting a solution

You need to follow the guideline in order to have successfully submitted solution. Currently we are supporting 3 types of user inputs. You need to write your code, assuming that user will enter one of these 3 types. It can be either an array of integers(1,2,3,4), it can be just a single integer(1 or 5.3) or it can be a function. Note:  if your case is the third one, you should be careful to parse the user input correctly. Here you can also find the examples.
Other points you should consider:
  • Name of the function must be the same both in input and in the code.
  • Your function must have exactly the same number of parameters as you mention in the number of parameters input
  • The names of parameters must be the same both in code and in the input
  • Test case input should be in the format, by which users are going to write it
  • Your function must take the test case input, process it and return the test case output
  • If the number of parameters is more than 1 you should split them with pipe |
Example:

Take an example of sum calculation

function calculateSum(set) {{ '{' }}
   const arr = set.split(',');
   const sum = arr.reduce((acc, cur) => acc + cur);
   return sum;
}
So you need to split user input as it will be comma separated numbers. After splitting you calculate the sum and return it.
